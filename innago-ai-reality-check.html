New Features — why AI works here
Feature work has well-defined scope, lives in known codebases, and follows clear patterns — exactly what AI tools excel at. Copilot and Claude are used daily for scaffolding, implementation, test generation, and iteration. This is where the 1.5–2× speed improvement is real. Accelerent shipped this way. PO Onboarding is nearing release. Refer & Earn is being built right now.
Support & Legacy — why AI can't reach this yet
Support and legacy bugs require tracing issues across multiple microservices, databases, and message queues — often spanning separate repos and domains. An engineer debugging a payment failure might trace from the API gateway, through RabbitMQ, into three different services, and down to a Snowflake data inconsistency. No AI tool can follow that full thread today. We're building the bridge — MCP for Freshdesk (POC tested) and agents for bug triage (in progress) give AI the connected context it needs.
Infra & DevSecOps — AI helps, but can't automate
Infrastructure work operates across live cluster state, config layers, cloud APIs, secrets, and runtime environments. AI assists with Terraform configs, runbooks, and pipeline scaffolding — but can't access live K8s state or navigate compliance boundaries. The Security Surveillance automation is the exception: it works because we built a specific MCP integration giving the agent direct Snowflake access. Replicating that pattern to more workflows is on the Q2 roadmap.

What's next
Expanding AI into more of what we do

The next phase focuses on bringing AI into the domains it can't yet reach — support triage, legacy bug work, and operational automation — while resolving the access and tooling gaps that are limiting broader adoption.

POC Tested · Nearing Production
MCP for Freshdesk — AI Support Triage
Agent integration for our support ticketing system. AI accesses ticket history, customer context, and resolution patterns. POC tested successfully — working toward production readiness. Targets the ~40% of engineering effort currently spent on support and legacy work.
In Progress · Target Q2 2026
Agents for Bug Triage & Data Fixes
Connecting AI agents to codebase context and database state so they can trace bugs across microservice boundaries and generate targeted fixes for data-layer issues. The bridge that makes legacy support work AI-accessible.
Target: April 2026
Standardized Toolchain + Full Licensing
Resolving provisioning blockers so every engineer has AI tools. Standardizing recommended tools per role. Removes the access barrier for the 35–40% of the team currently unable to fully experiment.
Q2 2026
Scale Operational Automation
Replicating the Security Surveillance and RabbitMQ patterns to 2–3 more toil-heavy workflows. Expanding AI-assisted operations beyond individual incidents to systematic, repeatable production automation.
